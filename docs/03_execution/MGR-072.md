# MGR-072 — DOCUMENT_OUTPUT_CENTER_CONTRACT

## Modalità

Docs-only / contract-first. Nessuna implementazione UI.

## Obiettivo

Definire il **Document Output Center** come area di controllo dei documenti prodotti, mantenendo il perimetro V1 locale:

- nessuna export reale aggiuntiva;
- nessun backend;
- nessun Google Drive;
- nessun OAuth;
- nessun DOCX/PDF programmatico;
- nessuna sincronizzazione remota;
- nessuna certificazione di approvazione istituzionale.

## File prodotti

- `docs/02_system/DOCUMENT-OUTPUT-CENTER-CONTRACT.md`

## Contenuto contratto

### Stati documento

Il contratto definisce sei stati output locale:

1. Bozza non modificata
2. Bozza modificata localmente
3. Salvato su questo dispositivo
4. Pronto per stampa
5. Da rivedere
6. Completato manualmente

### Azioni consentite

- Apri documento
- Continua bozza
- Stampa bozza
- Reset singolo documento
- Segna da rivedere
- Segna completato manualmente

### Azioni vietate per ora

- Export DOCX
- Export PDF programmatico
- Google Drive
- OAuth
- Backend
- Sincronizzazione remota
- Invio automatico
- Firma digitale
- Protocollazione

### Superfici coinvolte

- Vista Documenti
- Eventuale riepilogo Home
- Eventuale pannello “Output documenti”

### Persistenza

- Solo `localStorage`
- Nessun dato reale obbligatorio
- Nessuna persistenza remota
- Stato output separabile dalle bozze testuali

## Confini UX

Il contratto impone microcopy esplicito per evitare ambiguità:

- “Salvato su questo dispositivo”
- “Pronto per stampa locale”
- “Completato manualmente dall'utente”
- “Non equivale ad approvazione istituzionale”
- “Nessuna sincronizzazione remota”
