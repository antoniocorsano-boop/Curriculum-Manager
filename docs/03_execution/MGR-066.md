# MGR-066 — EDITABLE_DOCUMENT_DRAFT_MODEL

## Obiettivo

Introdurre un modello locale di documento editabile, usando la mappa MGR-065 come riferimento, senza ancora implementare il bridge automatico verso la matrice revisione e senza pannello laterale intelligente.

## Baseline

- MGR-064 chiuso remoto: DOCUMENT_SOURCE_IMPORT_EDITABLE_REVIEW_CONTRACT
- MGR-065 chiuso remoto: DOCUMENT_SOURCE_IMPORT_MAP
- HEAD allineato a origin/main
- working tree pulita
- app locale file:// funzionante

## File modificati

- `src/data/editableDocumentDraftModel.js` (nuovo)
- `src/views/documentiIstituzionaliView.js` (aggiornato)
- `index.html` (import nuovo JS statico)

## Modello dati implementato

### EditableDocumentStruttura
```javascript
{
  documentId: "curricolo-disciplina-campo",
  title: "Curricolo per disciplina/campo di esperienza",
  sourceId: "tecnologia-uda-base",
  sections: [
    {
      sectionId: "section-1",
      heading: "Dati generali",
      body: "[testo modificabile]",
      placeholder: "Istituto, anno, disciplina, ordine",
      order: 1,
      linkedRevisionArea: "Dati generali",
      outputRole: "contenuto modificabile"
    }
  ],
  version: "bozza",
  localDraftKey: "curriculumManager.editableDrafts.curricolo-disciplina-campo",
  updatedAt: "timestamp"
}
```

### Chiavi localStorage
- `curriculumManager.editableDrafts.<documentId>` - salva bozza documento
- `curriculumManager.editableDrafts` - prefisso generale

### Stati bozza
1. `nessuna-bozza` - nessun salvataggio precedente
2. `bozza-non-salvata` - modifiche ma non salvate
3. `bozza-salvata` - salvata su questo dispositivo

## Comportamenti verificati

- ✓ Aprire documento mostra sezioni editabili
- ✓ Textarea per ogni sezione con placeholder
- ✓ Pulsante "Salva bozza" salva in localStorage
- ✓ Pulsante "Ripristina bozza iniziale" pulisce modifiche
- ✓ Stato "Salvato su questo dispositivo" mostrato dopo salvataggio
- ✓ Bozza persiste al reload app
- ✓ Nessun auto-popolamento matrice revisione
- ✓ Nessun pannello laterale
- ✓ Badge "Fonte: SchoolDocs" per documenti mappati

## Vincoli rispettati

- ✓ No backend/API
- ✓ No auth/login
- ✓ No OAuth/Drive/Google
- ✓ No nuove dipendenze
- ✓ No fetch runtime di file locali
- ✓ No DOCX/PDF programmatico
- ✓ No dati studenti
- ✓ No dati personali nuovi
- ✓ No contenuti ufficiali inventati
- ✓ No sincronizzazione remota
- ✓ No bridge revisione in questa slice
- ✓ No pannello laterale in questa slice

## Rischi residui

1. **Persistencee browser**: Bozza si perde se localStorage pulito
2. **Cross-browser**: Nessun sincronismo tra browser
3. **Mobile**: Dimensioni textarea potrebbero necessitare aggiustamenti
4. **Section mapping**: I sectionId sono generati dinamicamente, potrebbero non essere stabili
