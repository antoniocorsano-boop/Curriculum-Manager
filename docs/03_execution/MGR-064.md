## Risposte alle domande obbligatorie

1. **Quali documenti reali o semi-reali sono già disponibili nei repo?**
   - SchoolDocs: 10 documenti con contenuto strutturato (~400-60 righe ciascuno), tutti con frontmatter metadata indicando `example_data: true`, `publishable: false`, `human_review_required: true`
   - Curriculum Manager: 10 documenti con solo struttura sezioni (titolo + description), senza corpo documento

2. **Quali documenti devono essere importati in Curriculum Manager?**
   - UDA template → mappato a "curricolo-disciplina-campo" o documento dedicato
   - Rubrica valutazione → mappato a "quadro-valutazione-rubriche" o documento dedicato
   - Griglia osservazione → mappato a nuovo tipo "strumento-osservazione"
   - Programmazione annuale → mappato a nuovo tipo "piano-annuale"
   - Verbale template → mappato a "documento-finale-dipartimento"
   - Schede revisione → mappato a "documento-revisione-aggiornamento"

3. **Quale formato interno deve avere un documento editabile?**
   - DocumentSource: metadati origine (id, title, type, originRepo, originPath, sourceStatus, isOfficial, humanValidationRequired)
   - EditableDocument: id documento, sourceId collegato, sections (array con body editabile), version, localDraftKey
   - DocumentSection: sectionId, heading, body (editabile), placeholder, order, linkedRevisionArea, outputRole
   - Tutti salvati in localStorage, nessun fetch runtime

4. **Come si distingue documento sorgente / bozza locale / modifica utente / revisione collegata / output finale?**
   - DocumentSource: template originale importato (immutabile, read-only)
   - Bozza locale: copia in localStorage con localDraftKey, modificabile
   - Modifica utente: DocumentChange registrato in localStorage, collegato a sezione
   - Revisione collegata: RevisionEntry generato automaticamente da DocumentChange
   - Output finale: esportazione JSON/Markdown da bozza consolidata

5. **Come viene salvata una modifica locale?**
   - DocumentChange creato con beforeText/afterText
   - Salvato in localStorage sotto `cmDocumentChanges-{documentId}`
   - Section body aggiornato in `cmEditableDoc-{documentId}`

6. **Come una modifica popola automaticamente la matrice revisione?**
   - Mutation observer su textarea change
   - Al salvataggio, crea RevisionEntry con sourceChangeId
   - RevisionEntry aggiunto a `cmRevisionEntries`
   - vista matriceRevisione mostra nuove voci con origine documento

7. **Come deve funzionare il visualizzatore laterale intelligente?**
   - Overlay/pannello slide-in non modale
   - Mostra: documento corrente, sezione selezionata, modifiche fatte, revisione collegata, stato locale
   - Persistenza stato in `SmartViewerState` localStorage
   - Non sostituisce vista principale, complementare
   - Toggle via icona apri/chiudi sempre disponibile

8. **Come evitare contenuti ufficiali inventati?**
   - Solo import da SchoolDocs con metadata `source_type: authored_internal`
   - Tutti i documenti mantengono `isOfficial: false`
   - Frontmatter obbligatorio con `human_review_required: true`
   - Nessun contenuto generato da AI o inventato

9. **Come mantenere compatibilità file://?**
   - Nessun fetch runtime
   - Tutti i dati in JS statico o localStorage
   - Nessun API esterna
   - Blob URL per export (già esistente)

10. **Quale slice implementativa deve venire dopo?**
    - MGR-065: DOCUMENT_SOURCE_IMPORT_MAP - mappare origine documenti
    - MGR-066: EDITABLE_DOCUMENT_DRAFT_MODEL - modello bozza
    - MGR-067: DOCUMENT_EDIT_TO_REVISION_BRIDGE - collegamento modifiche
    - MGR-068: SMART_DOCUMENT_VIEWER_PANEL - pannello laterale
    - MGR-069: OUTPUT_FROM_EDITABLE_DOCUMENTS - output

## Contratto UX

### Vista principale documento
- Titolo documento principale
- Editor sezioni inline (textarea per ogni sezione)
- Salvataggio locale automatico (debounce) o pulsante "Salva bozza"
- Azione "Segna per revisione" → crea RevisionEntry
- Azione "Apri pannello documento" → toggle SmartViewer
- Azione "Prepara output" → esporta JSON/Markdown

### Pannello laterale intelligente
- Sempre apribile via icona persistente
- Mostra documento corrente
- Mostra sezione selezionata
- Mostra modifiche fatte (ultime N)
- Mostra revisione collegata (se esiste)
- Mostra stato locale (bozza/conferma)
- Non sostituisce la vista principale
- Chiude con X o click overlay

### Matrice revisione (auto-popolata)
- Riceve automaticamente modifiche marcate "Segna per revisione"
- Non richiede ricopia manuale
- Mostra origine della modifica (documentId + sectionId)
- Permette completamento/conferma successiva
- Esportazione JSON/Markdown esistente mantiene funzionalità
