# MGR-074: Wiki interna dell'app

## Overview
Questa feature implementa una wiki interna integrata nell'applicazione Manager Curricolo d'Istituto. La wiki fornisce guida e aiuto per l'utilizzo dell'applicazione, spiegando le funzionalità realmente presenti senza riferimento a funzionalità mancanti o in sviluppo.

## Files Created/Modified
- `src/data/wikiCatalog.js` - Nuovo catalogo dati per i contenuti della wiki
- `src/views/wikiView.js` - Nuova vista per visualizzare i contenuti della wiki
- `index.html` - Aggiunta sezione wiki e import dei file necessari
- `src/app.js` - Aggiunta route per la wiki nella funzione renderView
- `src/components/sidebar.js` - Aggiunta voce "Wiki" nel menu laterale

## Feature Implemented
La wiki include le seguenti sezioni:
1. Che cos'è il Manager Curricolo
2. Documenti istituzionali: come leggere e compilare le bozze
3. Matrice revisione: come controllare completezza e coerenza
4. Output Center: cosa fa e cosa non fa
5. Validazione umana: criteri prima dell'uso ufficiale
6. Limiti e sicurezza: dati locali, nessun cloud

## UX Implementation
- Voce sidebar "Wiki" che apre la vista dedicata
- Vista Wiki leggibile subito con contenuti organizzati per categoria
- Struttura gerarchica: categorie → articoli
- Stile pulito basato sulle esistenti classi CSS dell'applicazione
- Link/CTA interni verso:
  * Documenti istituzionali
  * Matrice revisione
  * Output Center
- Stato vuoto gestito naturalmente (il catalogo contiene sempre contenuti)
- Copy prudente:
  * "consultazione interna"
  * "revisione umana richiesta"
  * "nessun invio automatico"
  * "nessun dato inviato a servizi esterni"

## Limits and Safety
- Frontend/static/local-only: tutta la wiki è contenuta nel client-side
- Nessun backend, API, OAuth, cloud coinvolti
- Nessuna esportazione automatica di documenti
- Nessun dato inviato a servizi esterni
- Spiegazione chiara che la validazione ufficiale deve avvenire secondo le procedure istituzionali
- Nessuna promessa di ufficialità attraverso l'applicazione
- Nessun riferimento a funzionalità non presenti (es. DOCX programmatico senza gate)

## Validation
- Tutti i file JavaScript passano i controlli di sintassi (node --check)
- Nessun riferimento a funzionalità vietate (fetch, XMLHttpRequest, etc.)
- Implementazione basata esclusivamente sulle funzionalità realmente presenti
- Rispetto delle convenzioni di codifica esistenti nell'applicazione
